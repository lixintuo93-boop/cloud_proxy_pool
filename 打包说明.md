# cloud_proxy_pool 打包说明

把 `gui_app.py` 打包成单文件 `cloud_proxy_pool.exe`，拷到其它 Windows 机器双击即可运行（不需要装 Python）。
spec 文件 `cloud_proxy_pool.spec` 已经写好，直接按下面流程跑就行。

---

## 1. 准备环境（只需一次）

在项目目录下执行：

```powershell
# 装 PyInstaller
pip install pyinstaller

# 装项目运行依赖（如果还没装过）
pip install paramiko
```

可选：装 UPX 让 EXE 体积小 30%~50%。从 https://upx.github.io 下载 `upx.exe`，丢进任意 PATH 目录（比如 `C:\Windows\`）。没装会自动忽略，不报错。

---

## 2. 打包前：同步 gamyy-core 到 resources/

EXE 会把 `resources/gamyy_core/` 整个目录内置进去。打包前要确保这里是你想要的版本。

**两种方式任选其一：**

**方式 A（推荐）**：跑一次 GUI，进 "Agent 部署"页 → 点 **📥 从外部同步**。
它会从 `config.py:129` 里的 `AGENT_SOURCE_DIR`（当前指向 `E:\gamyy_base_info\gamyy-core-20260420-1`）同步过来，自动按完整模式排除规则保留 `data/config.db`、`data/hospital.db`。

**方式 B（手动）**：

```powershell
# 先清掉旧的
Remove-Item -Recurse -Force resources\gamyy_core -ErrorAction SilentlyContinue

# 再整个拷过来
Copy-Item -Recurse E:\gamyy_base_info\gamyy-core-20260420-1 resources\gamyy_core
```

打包**不需要** gamyy-core 也可以（spec 里只会 WARN），但产物 EXE 第一次部署 agent 时会找不到内置源。

---

## 3. 打包

项目根目录下执行：

```powershell
pyinstaller cloud_proxy_pool.spec
```

第一次大约 1~3 分钟。结束后产物在：

```
dist\cloud_proxy_pool.exe
```

体积大概 30 MB 左右（含 gamyy_core）。直接双击就能跑，可以拷到其它 Windows 机器使用。

---

## 4. 清理 & 重新打包

源码改了重新打包：直接重跑 `pyinstaller cloud_proxy_pool.spec` 即可，会自动覆盖 `dist\`。

如果想彻底干净地重打（比如换了 Python 版本、依赖、或者怀疑缓存有问题）：

```powershell
Remove-Item -Recurse -Force build, dist
pyinstaller cloud_proxy_pool.spec
```

`build\` 是中间产物，可以随时删；`dist\` 是最终产物。

---

## 5. gamyy-core 改了之后怎么办？

两种思路：

- **方案 A**：重新执行第 2 步同步，再重新打包，发布新 EXE。
- **方案 B**：让现有 EXE 用户在 GUI 里点 **📤 导入 zip**，热替换内置 gamyy-core，不用重新发 EXE。

日常小改动用 B，发版本用 A。

---

## 6. spec 关键配置说明

`cloud_proxy_pool.spec` 里几处可能需要改的：

| 位置 | 作用 | 默认 |
|---|---|---|
| `console=False` | GUI 程序不弹黑窗 | False（保持） |
| `upx=True` | 用 UPX 压缩 | True |
| `icon='icon.ico'` | EXE 图标 | 注释掉的，想加图标就放个 `.ico` 文件然后取消注释 |
| `hiddenimports` | 显式声明的隐式导入 | 已包含 paramiko 全家桶 |
| `excludes` | 排除大模块 | matplotlib/numpy/pandas/scipy/各种 Qt |

---

## 7. 常见问题

**Q：双击 EXE 闪退？**
临时把 spec 里 `console=False` 改成 `console=True`，重新打包，就能看到报错信息。排查完改回来。

**Q：报错 "ModuleNotFoundError: No module named 'xxx'"？**
在 spec 的 `hiddenimports` 里加上 `'xxx'`，重新打包。

**Q：EXE 跑起来但找不到 `resources/...` 里的文件？**
PyInstaller 单文件模式下，资源被解压到临时目录 `sys._MEIPASS`。代码里读资源得用 `config.py` 里的 `_app_root()` 之类的辅助方法，别直接用相对路径。

**Q：打包很慢/EXE 很大？**
检查 `excludes` 是否漏掉了不用的大模块；确认没有意外把 venv 或 `__pycache__` 卷进 `resources/`。

**Q：杀毒软件报毒？**
PyInstaller 打的 EXE 经常被误报。给打包后的 EXE 加代码签名，或者临时把它加入杀软白名单。

## 8. 一键命令汇总

完整发版流程：

```powershell
# 1. 同步 gamyy-core（也可以走 GUI 同步按钮）
Remove-Item -Recurse -Force resources\gamyy_core -ErrorAction SilentlyContinue
Copy-Item -Recurse E:\gamyy_base_info\gamyy-core-20260420-1 resources\gamyy_core

# 2. 清掉旧产物
Remove-Item -Recurse -Force build, dist -ErrorAction SilentlyContinue

# 3. 打包
pyinstaller cloud_proxy_pool.spec

# 4. 产物
explorer dist
```
